# 🤖 RosterIQ: Autonomous Pipeline Intelligence
**The Strategic Command Center for Healthcare Roster Operations**

RosterIQ is a high-performance, autonomous AI agent designed to eliminate bottlenecks in healthcare provider roster processing. Built for the **HiLabs AI Hackathon (Cognizance 2026)**, it transforms complex operational data into actionable surgical interventions.

---

## 🌟 Core Pillars

### 1. Transparent Memory Architecture
Unlike stateless bots, RosterIQ features a **Visible Layered Memory** system:
- **🗂️ Episodic Memory**: Context-aware retrieval of past investigations (e.g., "KS Market failures from 2 sessions ago").
- **⚙️ Procedural Memory**: Standardized execution of complex sub-routines (e.g., `market_health_report`).
- **🧬 Semantic Context**: Deep injection of domain knowledge (AvailityPDM, Medicaid FFS Compliance, NPI validation rules).

### 2. High-Fidelity Strategic Dashboards
Five mandatory visualizations designed for maximum visual/AI alignment:
- **🔴 Pipeline Stage Heatmap**: High-fidelity grid with **inner-cell G/Y/R labels** for worst-case monitoring.
- **📊 Duration Anomaly Tracker**: Side-by-side actual vs. benchmark metrics with automated latency highlighting.
- **📈 Market SCS% Stability**: Success trends vs. a strict **85% SLA Threshold**.
- **♻️ Retry Lift Analysis**: Quantification of the autonomous repair pipeline's effectiveness.
- **🗄️ Stuck RO Tracker**: Real-time escalation desk for trapped rosters.

### 3. Actionable AI (Findings ➔ Action)
Every insight generated by RosterIQ follows a strict surgical pattern:
1. **Finding**: Data-driven observation.
2. **Evidence**: Specific quantification (e.g., "30.7% lift").
3. **Recommended Action**: A concrete next step (e.g., "Pause automated loading for Org X").

## 🛠️ Technology Stack
- **Engine**: DuckDB (Fast SQL Analytics), Pandas, NumPy.
- **Intelligence**: LangChain, OpenRouter (Llama 3 / Gemini), Tavily (Real-time Compliance Search).
- **Memory**: ChromaDB (Vector store for long-term audit trails).
- **UI**: Streamlit (High-Fidelity Interaction Design).

## 🚀 Quick Start

1. **Clone & Setup Environment**:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Windows: .\venv\Scripts\activate
   pip install -r requirements.txt
   ```

2. **Configure API Keys**:
   Create a `.env` file (see `.env.example`):
   ```env
   OPENROUTER_API_KEY=your_key
   TAVILY_API_KEY=your_key
   ```

3. **Launch the Command Center**:
   ```bash
   streamlit run app.py
   ```
